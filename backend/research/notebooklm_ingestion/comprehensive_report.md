## 2. Intelligence Orchestration: Task-Based Modalities
Efficiency is achieved by matching computational primitives to task complexity through specialized operating modes.

### Fast Mode: Low-Latency Execution
- **Optimized for:** Gemini 3 Flash / Gemini 3 Pro (Low).
- **Use Case:** Code completion, variable renaming, and routine terminal commands.
- **Mechanism:** Direct execution without intensive planning cycles.

### Planning Mode: Deep Reasoning
- **Optimized for:** Gemini 3 Pro (High Reasoning) / Claude 4.5.
- **Use Case:** Architectural shifts, complex feature builds, and refactoring.
- **Mechanism:** Triggers a "Deep Research" cycle that produces structured **Artifacts** for human approval before execution.

### Loki Mode: Autonomous Swarming
- **Mechanism:** The peak of agentic coordination where specialized agents (Developer, Tester, Security) work in parallel across the Editor, Terminal, and Browser.

## 4. Knowledge Grounding: NotebookLM & MCP
Grounding is the primary defense against LLM hallucinations. The engine utilizes the **Model Context Protocol (MCP)** as a "USB-C for AI" to connect agents to proprietary knowledge.

- **Grounding Mechanism:** Using the `@notebooklm` skill to query project-specific documents (PDFs, docs, brand guides).
- **Agent Skills:** Modular `.md` "micro-brains" (e.g., `react-components`, `vercel-deploy`) stacked to provide agents with domain-specific expertise.
- **Technical Implementation:** Skills are symlinked to `~/.gemini/antigravity/skills` for instant global synchronization across workspaces.
